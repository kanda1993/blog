exportされた会話ログファイルを記事用のlog.mdに変換してください。

ログファイル: $ARGUMENTS

## 手順

### 1. 対象の記事ディレクトリを確認

どの記事に紐づけるか聞いてください。

### 2. スクリプトで基本変換を実行

```bash
docker compose exec app bun run convert-log <ログファイル> <記事ディレクトリ>
```

※ コンテナが起動していない場合は先に `docker compose up -d` を実行

スクリプトが行うこと：

- JSONLの構造解析
- Markdown形式への整形（User: 引用ブロック / Assistant: 通常テキスト）
- 基本的なフォーマット処理

### 3. AIで補正作業

生成された `log.md` を読み込み、以下の補正を行う：

#### セキュリティ・プライバシー

- **パスのマスキング**: `/Users/実名/...` → 匿名化（サンプルパスは残してOK）
- **公開すべきでない情報の除去**: 認証情報、APIキー、内部URL等

#### 編集指示のやり取りを削除

記事作成プロセスに関するやり取りは学習内容ではないため削除：

- **記事作成の冒頭部分**: スラッシュコマンド実行、テンプレート作成、「どこから学びたいですか？」等
- **記事への追加指示**: 「この内容を記事に追加しますか？」「記事に追加して」「記事に反映して」とその応答
- **編集指示**: 「mermaidにしてください」「折りたたんで」「修正して」とその応答
- **次の質問の促し**: 「次は何を学びますか？」
- **システムメッセージ**: `<command-message>`, `<command-name>`, `Caveat:` 等

#### フォーマット確認

- **コードブロック**: 正しくフォーマットされているか
- **不要なメタ情報**: ツール実行結果の冗長な部分

### 4. 最終確認

修正後の内容をユーザーに確認してもらう
